## Code Blocks, Indentation & Statements in Java

---

## Code Blocks in Java

A **code block** is a group of Java statements enclosed within  
**curly braces `{ }`**. Code blocks define the **scope** of variables  
and group logically related statements.

### Types of Code Blocks
- **Class Block**
- **Method Block**
- **Loop Block** (`for`, `while`, `do-while`)
- **Conditional Block** (`if`, `else`, `switch`)
- **Standalone Block**

### Example
```java
class Example {
    void show() {          // Method block
        int x = 10;        // Block scope variable
        if (x > 5) {       // Conditional block
            System.out.println(x);
        }
    }
}
```

✅ Variables declared inside a block **cannot be accessed outside** that block.

---

## Indentation in Java

**Indentation** refers to the use of spaces or tabs to organize code  
visually. Java does **not enforce indentation**, but it is critical for  
**readability and maintenance**.

### Standard Indentation Rules
- Use **2 or 4 spaces** per level (4 spaces is common)
- Indent code inside:
    - Classes
    - Methods
    - Loops
    - Conditionals

### Proper Indentation Example
```java
if (age >= 18) {
    System.out.println("Eligible to vote");
} else {
    System.out.println("Not eligible");
}
```

### Poor Indentation (Avoid)
```java
if(age>=18){
System.out.println("Eligible");
}else{
System.out.println("Not eligible");
}
```

✅ Proper indentation makes code **easy to understand** and **debug**.

## Statements in Java

A **statement** is a **complete unit of execution** in Java.  
Most Java statements **end with a semicolon (`;`)**.

---

## Types of Statements in Java

### 1. Declaration Statement
Used to declare variables.

```java
int age;
double salary;
```

---

### 2. Assignment Statement
Used to assign values to variables.

```java
age = 25;
salary = 50000.0;
```

---

### 3. Expression Statement
Performs computation or method calls.

```java
int sum = a + b;
System.out.println(sum);
```

---

### 4. Control Flow Statements
Used to control the execution flow.

#### Conditional Statements
```java
if (marks >= 40) {
    System.out.println("Pass");
} else {
    System.out.println("Fail");
}
```

#### Loop Statements
```java
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}
```

---

### 5. Jump Statements
Used to alter the flow abruptly.

```java
break;
continue;
return;
```

Example:
```java
if (x < 0) {
    return;
}
```

---

## Semicolon (`;`) in Java
- Ends a **statement**
- Not required after **code blocks**

✅ Correct:
```java
int x = 10;
```

❌ Incorrect:
```java
if (x > 5);
```

---

## Key Points to Remember
- Code blocks define **scope**
- Indentation improves **readability**, not execution
- Statements represent **actions**
- Most statements end with a **semicolon**
- Clean formatting leads to **maintainable code**
