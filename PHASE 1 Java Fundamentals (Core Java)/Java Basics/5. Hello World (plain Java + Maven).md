## **Hello World (Plain Java + Maven)**

---
### **Overview**
A *Hello World* program using **Plain Java with Maven** explains how a simple  
Java application is structured, built, and executed using Maven’s standard  
project conventions.

---

### **Plain Java**
- Plain Java means a **basic Java program** without using any frameworks.
- The program contains a **class** and a **main() method**.
- Execution of the program always starts from the **main() method**.

---

### **Maven**
- **Maven** is a build automation and dependency management tool.
- It standardizes:
    - Project structure
    - Build process
    - Dependency handling
- Maven uses a configuration file called **`pom.xml`** (Project Object Model).

---

### **Maven Project Structure**
A standard Maven project follows this structure:

- `src/main/java` – Application source code
- `src/test/java` – Test source code
- `pom.xml` – Project configuration and build instructions

This standard layout helps tools, IDEs, and build systems understand projects  
without additional configuration.

---

### **Role of `pom.xml`**
The **`pom.xml`** file defines:
- Project identity (**groupId, artifactId, version**)
- Packaging type (JAR in this case)
- Java version for compilation
- Build lifecycle rules

Maven reads this file to manage and control the entire build process.

---

### **Execution Flow**
1. The developer writes Java source code (`.java` file).
2. Maven compiles the source code using the **Java compiler (`javac`)**.
3. The compiler converts source code into **bytecode (`.class` files)**.
4. Maven packages the compiled bytecode into a **JAR file**.
5. The **JVM** loads the JAR file and starts execution from the `main()` method.
6. The program prints output to the console.

---

### **Why Use Maven for Hello World**
- Follows **industry-standard project layout**
- Ensures **platform-independent builds**
- Makes projects **scalable** for future enhancements
- Prepares developers for **enterprise-level Java development**

---

### **Output**
When executed, the program displays:
``` 
Hello, World!
```
---

### **Conclusion**
- **Plain Java** handles application logic
- **Maven** manages the build lifecycle
- Together, they provide a clean, maintainable, and scalable setup for Java  
  applications
